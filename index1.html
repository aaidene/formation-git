Principe commande git

1.Les premier commit

git init : Initialiser un nouveau dépôt Git

                        
        
                 git init # Initialiser un nouveau dépôt Git

status emvérifier l'état du versioning

        
        git status  # Vérifier l'état du versioning

Add:staging : Sélectionner les fichiers pour le prochain commit

        
        git add fichier  # Ajouter un fichier spécifique au staging
        git add   # Ajouter tous les fichiers d'un dossier au staging
        git add *.html  # Ajouter tous les fichiers HTML au staging
        git add --all  # Ajouter tous les fichiers (ajouts et suppressions) au staging
        git add *  # Ajouter tous les fichiers (ajouts et suppressions) au staging

commit:effectuer le premier commit

        
        git commit  # # Ouvrir un éditeur pour rédiger le message du commit
        git commit -m "Message pour le commit ^^"  # Utiliser le drapeau -m pour spécifier directement le message du commit

logconsulter l'historique des commits

            
            git log  # Récupère et affiche les derniers commits
            git log --oneline # Afficher l'historique avec une ligne par commit (plus lisible)
            git log -n   # Afficher un nombre spécifié de commits
            git log -p   # Afficher l'historique des commits affectant un fichier en particulier
            git log --author   # Afficher l'historique par rapport au nom de l'auteur

diff:Voir les différences sur un fichier

            
            git diff # Afficher toutes les différences depuis le dernier commit
            git diff    # Afficher les différences pour un fichier spécifique
            git diff --cached  # Afficher les différences par rapport à ce qui est en staging
            git diff   # Comparer l'état actuel au commit spécifié
            git diff ..   # Comparer les fichiers entre deux commits

checkout: Passer de branche en branche ou revenir sur un fichier ou un commit

                
                    git checkout  # Transformer le fichier comme il était lors du commit et l'ajouter au staging
                    git checkout   # Transformer tous les fichiers pour reproduire l'état du commit. brMet le HEAD dans un état détaché (detached HEAD)

revert: Inverser un commit

                
                    git revert    # Inverser un commit en créant un nouveau commit d'inversion

reset: Altérer l'historique et la zone de staging

                
                    git reset fichier> # Supprimer un fichier de la zone de staging sans supprimer les modifications
                    git reset   # Supprimer tous les fichiers de la zone de staging sans supprimer les modifications
                    git reset --hard # Réinitialiser le dossier de travail au dernier commit. Toutes les modifications non commit seront perdues
                    git reset   # Revenir en arrière jusqu'au commit. Réinitialise la zone de staging tout en laissant le dossier de travail inchangé. L'historique sera perdu
                    git reset  --hard   # Revenir au commit et réinitialiser la zone de staging et le dossier de travail pour correspondre. Toutes les modifications et les commits après le commit seront supprimés
